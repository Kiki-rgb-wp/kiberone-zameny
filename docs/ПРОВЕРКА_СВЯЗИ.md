# Проверка связи фронта с бэкендом и бота

## 1. URL бэкенда (как на скрине Railway)

Используй **ровно этот** адрес (без слэша в конце):

```
https://earnest-beauty-production-583b.up.railway.app
```

- В **Vercel** → Settings → Environment Variables переменная **VITE_API_URL** должна быть равна этой строке.
- После изменения переменной — **Redeploy** проекта (Deployments → ⋯ → Redeploy, при необходимости «Clear build cache»).

## 2. Подсказка в приложении

Если при нажатии «Войти» видишь красный текст:

- **«Не задан URL API. На Vercel добавьте переменную VITE_API_URL…»**  
  → значит при сборке на Vercel переменная не подставилась. Добавь **VITE_API_URL**, сохрани и сделай **Redeploy**.

- **«Нет связи с сервером. Проверьте, что бэкенд на Railway запущен и URL верный.»**  
  → значит URL задан, но запрос до бэкенда не доходит. Проверь п. 3 и 4.

## 3. Проверка бэкенда в браузере

Открой в браузере (телефон или ПК):

```
https://earnest-beauty-production-583b.up.railway.app
```

- Страница открылась (пустая, «Cannot GET /» или текст) → бэкенд работает.
- Не открывается / таймаут → бэкенд недоступен (инцидент Railway или сервис спит). Подожди и проверь снова.

## 4. Бот в Telegram

- В BotFather в **Main App** или **Menu Button** указан **URL фронта** с Vercel (например `https://kiberone-zameny.vercel.app`), **не** URL Railway.
- Открываешь бота → кнопка меню → открывается форма «Замены KIBERone». Запросы с этой страницы идут на URL из **VITE_API_URL** (Railway).

## Итог

| Что проверить | Где | Ожидание |
|---------------|-----|----------|
| URL бэкенда | Vercel → Environment Variables → VITE_API_URL | `https://earnest-beauty-production-583b.up.railway.app` |
| Пересборка после смены переменной | Vercel → Deployments → Redeploy | Новый деплой успешен |
| Работа бэкенда | Браузер → открыть URL выше | Страница открывается |
| URL в боте | BotFather → Main App / Menu Button | URL фронта Vercel |
